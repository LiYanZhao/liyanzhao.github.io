JNI概述

Java Native Interface简称JNI，即Java本地调用，JNI定义了一种Java层与 Native层通信方式，能使我们在Java层调用Native函数（C/C++函数），同时也能使我们在Native层调用Java方法。对我们开发人员来说丰富成熟的Native语言模块及出色的运行效率能使我们的开发工作更快捷，更有效率。

JNI方法注册

JNI方法注册分为静态注册与动态注册，我们上面开发流程所使用的方式为静态注册，静态注册根据函数名称查找，Native函数名称必须包含完整的类名，否则在调用时就无法找到对应的Native函数。下面我们主要来实现动态注册。

lookupSharedLibMethod是从我们加载的so库中查找函数，gDvm类型为DvmGlobals,存储与整个dalvik相关的参数及成员变量，dvmHashForeach这个函数主要是循环遍历gDvm.nativeLibs，然后调用findMethodInLib去查找method：

findMethodInLib内做了两次查找（dlsym），第一次查找是根据类名 + 函数名 查找，如果第一次没找到，第二次则再加上签名信息进行查找。Method找到后让我们回到dvmResolveNativeMethod,接着会调用dvmUseJNIBridge：

JNI方法并不是直接调用，而是通过Dalvik虚拟机间接调用，间接调用JNI方法的函数称为Bridge，Bridge函数在调用JNI方法前会执行一些通用的初始化工作，dvmUseJNIBridge主要是通过Dalvik虚拟机的启动选项选择一个合适的Bridge函数，然后调用dvmSetNativeFunc将其赋值给Method的nativeFunc. dvmUseJNIBridge调用完成后下一步就是直接调用Method的nativeFunc所指向的函数，也就是Bridge函数，间接完成我们的JNI方法的调用。
以上是静态注册的调用流程，那么动态注册的有何区别 ？ 我们先来看看动态注册都做了什么：

注册的过程实际上是查找Native函数，并调用dvmUseJNIBridge完成nativeFunc赋值的过程，将静态注册方式执行阶段的工作提前到加载库时完成，因此动态注册不仅在函数名称上方便定义，在执行效率上也是略优于静态注册。
