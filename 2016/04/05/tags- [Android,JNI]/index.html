JNI概述

Java Native Interface简称JNI，即Java本地调用，JNI定义了一种Java层与 Native层通信方式，能使我们在Java层调用Native函数（C/C++函数），同时也能使我们在Native层调用Java方法。对我们开发人员来说丰富成熟的Native语言模块及出色的运行效率能使我们的开发工作更快捷，更有效率。

JNI开发流程

用Android Studio做JNI开发，首先在local.properties中配置NDK目录

  1. ndk.dir= ${ndk local dir}

然后还需要在gradle.properties中配置

  1. android.useDeprecatedNdk=true

环境配置好后，我们还需要在build.gradle中配置生成的动态库的信息

  1. defaultConfig {
  2.     ndk{
  3.         moduleName="helloJni" //生成的so库名称
  4.         // abiFilters "armeabi","x86" //指定支持的ABIs
  5.     }
  6. }

现在我们开始定义Native函数

  1. public class JniTest {
  2.     static {
  3.         System.loadLibrary("helloJni");  //加载动态库
  4.     }
  6.     public static native String helloJni(JniTest test);
  8.     public String nativeCallJava(){
  9.         return "native call java";
  10.     }
  11. }

helloJni就是我们定义的Native函数,定义好后helloJni会显示红色，alt + enter 查看会提示

点击Create function，Android Studio会自动帮我们创建jni/helloJni.c

  1. #include <jni.h>
  3. JNIEXPORT jstring JNICALL
  4. Java_test_jni_JniTest_helloJni(JNIEnv *env, jclass type, jobject test) {
  5.     // TODO
  6.     return (*env)->NewStringUTF(env, "hello jni");
  7. }

现在我们就可以直接调用helloJni这个Native函数了，Android Studio会自动帮我们生成libhelloJni.so，不用我们再去手动ndk-build了。

JNI函数调用

在上面我们知道了JNI的整个开发流程，已经能准确的调用我们自己所写的Native函数helloJni,可是我突然想知道我们在Java中调用helloJni时是如何调用到Java_test_jni_JniTest_helloJni这个Native函数上去的，他们之间是如何做的关联。

dalvik中调用函数则会调用dvmCallMethod(/dalvik/vm/interp/Stack.cpp),在dvmCallMethod内部实际调用的是dvmCallMethodV，我们来看看在dvmCallMethodV中对Native函数是如何调用的:

  1. if (dvmIsNativeMethod(method)) {
  2.      TRACE_METHOD_ENTER(self, method);
  3.      (*method->nativeFunc)((u4*)self->interpSave.curFrame, pResult,method, self);
  4.      TRACE_METHOD_EXIT(self, method);
  5. } else {
  6.      dvmInterpret(self, method, pResult);
  7. }

当Method是Native函数时则调用method->nativeFunc所指向的函数，那么Method的nativeFunc指向哪里？ 又是在什么地方赋值的呢 ？是否在类加载时被解析赋值 ? 下面我们看下类的加载。

以上是类的加载序列图，一路跟踪下来，我们跟踪到loadMethodFromDex这个函数，从函数名称就看的出来这个函数是用来从Dex文件解析Method，我们现在主要关心的是Method的nativeFunc，让我们来看看loadMethodFromDex内有关nativeFunc的部分：

  1.  if (dvmIsNativeMethod(meth)) {
  2.      meth->nativeFunc = dvmResolveNativeMethod;
  3.      meth->jniArgInfo=computeJniArgInfo(&meth->prototype);
  4.   }

果然，Method的nativeFunc是在类加载时被赋值，指向dvmResolveNativeMethod这个函数，不过所有的Native函数都指向dvmResolveNativeMethod，很明显dvmResolveNativeMethod这个函数是个通用函数，仍然未能调用到我们所写的Native函数，我们接着看dvmResolveNativeMethod的源码：
